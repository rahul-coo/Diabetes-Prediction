Configure CORS on FastAPI
Critical fix
Your SOP omits CORS setup — React on localhost:5173 calling FastAPI on :8000 will be blocked. 
from fastapi.middleware.cors import CORSMiddleware
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_methods=["*"],
    allow_headers=["*"],
)

7
PredictionForm component
Core
Uses react-hook-form for validation — avoids messy useState per field: 
import { useForm } from 'react-hook-form'

const fields = [
  { name: 'Pregnancies', min: 0, max: 20 },
  { name: 'Glucose', min: 0, max: 300 },
  { name: 'BloodPressure', min: 0, max: 200 },
  { name: 'SkinThickness', min: 0, max: 100 },
  { name: 'Insulin', min: 0, max: 900 },
  { name: 'BMI', min: 0, max: 70, step: 0.1 },
  { name: 'DiabetesPedigreeFunction', min: 0, max: 2.5, step: 0.01 },
  { name: 'Age', min: 1, max: 120 },
]

export default function PredictionForm({ onSubmit }) {
  const { register, handleSubmit, formState: { errors } } = useForm()
  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      {fields.map(f => (
        <div key={f.name}>
          <label>{f.name}</label>
          <input type="number" step={f.step || 1}
            {...register(f.name, { required: true, min: f.min, max: f.max })} />
          {errors[f.name] && <span>Invalid value</span>}
        </div>
      ))}
      <button type="submit">Predict</button>
    </form>
  )
}

10
Axios API module
Required
Your SOP shows a one-liner — this is the proper production version: 
// src/api/predictApi.js
import axios from 'axios'

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  timeout: 10000,
})

export const predict = async (formData) => {
  const res = await api.post('/predict', formData)
  return res.data  // { result: 0|1, probability: 0.87 }
}

export const healthCheck = async () => {
  const res = await api.get('/health')
  return res.data
}
11
FastAPI response shape
Align this
Your SOP backend returns {"result": model.predict([data])} — extend it to include probability for the UI confidence display: 
@app.post("/predict")
def predict(data: PatientData):
    features = [[data.Pregnancies, data.Glucose, ...]]
    result = int(model.predict(features)[0])
    prob = float(model.predict_proba(features)[0][result])
    return {"result": result, "probability": prob}
12
Custom hook — usePrediction
Clean pattern
Keeps Dashboard.jsx clean — all async logic in one hook: 
// src/hooks/usePrediction.js
import { useState } from 'react'
import { predict } from '../api/predictApi'

export function usePrediction() {
  const [result, setResult] = useState(null)
  const [loading, setLoading] = useState(false)
  const [error, setError] = useState(null)
  const [history, setHistory] = useState([])

  const submit = async (formData) => {
    setLoading(true); setError(null)
    try {
      const data = await predict(formData)
      setResult(data)
      setHistory(prev => [...prev, { ...formData, ...data, time: new Date() }])
    } catch (err) {
      setError(err.response?.data?.detail || 'Prediction failed')
    } finally {
      setLoading(false)
    }
  }
  return { result, loading, error, history, submit }
}

Dark mode context

// src/context/ThemeContext.jsx
import { createContext, useContext, useState } from 'react'
const ThemeCtx = createContext()

export function ThemeProvider({ children }) {
  const [dark, setDark] = useState(false)
  return (
    <ThemeCtx.Provider value={{ dark, toggle: () => setDark(p => !p) }}>
      <div className={dark ? 'dark' : ''}>{children}</div>
    </ThemeCtx.Provider>
  )
}
export const useTheme = () => useContext(ThemeCtx)
In Tailwind config add: darkMode: 'class'

}

16
Vite proxy — avoid CORS in dev
Alternative
Instead of CORS middleware, configure Vite to proxy: 
// vite.config.js
export default {
  server: {
    proxy: {
      '/predict': 'http://localhost:8000',
      '/health': 'http://localhost:8000',
    }
  }
}
Then use axios.post('/predict', data) with no base URL needed in dev.
!
Issues found in your SOP + fixes
Review all
Missing CORS config — React on port 5173 calling FastAPI on 8000 will fail with a CORS error. Add CORSMiddleware to your FastAPI app (see Step 3).
Backend predict returns array — model.predict([data]) returns a numpy array, not a scalar. Use int(model.predict([data])[0]). Also return predict_proba for confidence scores.
No input schema on FastAPI — the SOP shows def predict(data) with no type. Use a Pydantic model so FastAPI validates input automatically.
Zero values in Pima dataset — Glucose, BloodPressure, SkinThickness, Insulin, BMI cannot physiologically be zero. Your SOP mentions this but doesn't enforce it on the frontend. Add min validation in react-hook-form (e.g. Glucose min=1).
No scaler saved — if you normalize features before training, you must save the scaler too: joblib.dump(scaler, 'scaler.pkl') and apply it in the API before prediction.
Suggestion: Add a /health endpoint — lets the frontend show a "backend offline" warning gracefully if FastAPI is down.
Suggestion: Use Pydantic for input — class PatientData(BaseModel): Pregnancies: int; Glucose: float ... gives automatic validation and docs at /docs.
